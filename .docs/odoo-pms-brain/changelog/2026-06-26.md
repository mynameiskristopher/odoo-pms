# Changelog - 2026-06-26

## [Added] Owner and Unit Ownership Models

- Created `pms.owner` model representing property owners (companies/individuals) with full 1099 tax, ACH payment, insurance, travel agent commission, and financial details.
  - Inherits `mail.thread` for chatter/followers support.
  - Uses Odoo's built-in `active` field for archive/unarchive (not custom `is_active`).
  - Uses Odoo's built-in audit fields (`create_date`, `write_date`, `create_uid`, `write_uid`) instead of custom timestamp fields.
  - Fields include: identity, primary address, tax address, 1099/tax info, ACH/payment, insurance (deprecated), travel agent commission, work order approval, contact details, tags, financial balances.
  - Constraints on: travel agent commission (0â€“100), ACH routing number (9 digits), country codes (2-char ISO).
- Created `pms.owner.tag` model for tagging owners with color support (for `many2many_tags` widget).
- Created `pms.unit.owner` model linking owners to properties/units with contract info, dates, and fractional ownership support.
  - Computed `owner_name` and `unit_name` fields via `related=` for list view display.
  - Date validation constraint (end date â‰Ą start date).
- Added `unit_owner_ids` One2many field on `pms.property` for ownership history from the property side.
- Created full XML views (list, form, search) for both models.
- Added "Ownership" tab to the property form view.
- Added ACLs for `pms.owner`, `pms.owner.tag`, `pms.unit.owner` in `ir.model.access.csv`.
- Updated `models/__init__.py` and `__manifest__.py` to register new models and views.

## [Fixed] Odoo Convention Errors Blocking Module Install

- Removed `contact_ids` One2many that referenced non-existent `pms_owner_id` field on `res.partner` (was causing `KeyError: 'pms_owner_id'`).
- Replaced custom `is_active` boolean with Odoo's built-in `active` field pattern.
- Replaced custom audit fields (`created_at`, `created_by`, `updated_at`, `updated_by`) with Odoo ORM magic fields (`create_date`, `create_uid`, `write_date`, `write_uid`).
- Added `color` field to `PmsOwnerTag` for `many2many_tags` widget with `color_field`.
- Replaced `<div class="oe_chatter"><field name="message_ids"/></div>` with Odoo 18's `<chatter/>` tag.
- Added `mail` to manifest dependencies for `mail.thread` inheritance.
- Fixed malformed domain syntax in unit owner search view (`|` OR operator must use prefix notation inside domain list).
- Changed ACH/Payment view group from non-existent `pms.group_pms_manager` to `base.group_system`.
- Added `currency_id` computed field and invisible column in list views for monetary widget support.

## [Removed] Fractional Ownership Removed

- Removed `fractional_inventory_id` field from `pms.unit.owner` model.
- Removed `contract_type` field from `pms.unit.owner` model (only had `standard` and `fractional` options; now unnecessary).
- Removed `fractional_inventory_id` and `contract_type` from unit owner list, form, and search views.
- Removed `contract_type` from the owner form's Unit Ownership tab.
- Removed `contract_type` from the property form's Ownership tab.
- Removed "Fractional" search filter and "Contract Type" group-by from unit owner search view.

## [Added] Hierarchical Zones and Customizable Zone Types

- Created `pms.zone.type` model to allow customizable classifications of zones (e.g. Region, Building, Floor).
- Created `pms.zone` model to represent hierarchical zones, utilizing Odoo's native `_parent_store` and `complete_name` patterns for nested paths.
- Added `zone_id` Many2one field to `pms.property` model to link properties to their corresponding parent zone.
- Added views and menus:
  - Created list, form, and search views for `pms.zone` and `pms.zone.type` in `views/pms_zone_views.xml`.
  - Added a new top-level "Configuration" category to the main PMS menu.
  - Placed "Zone Types" under Configuration and "Zones" under the Properties menu.
- Updated `pms.property` views:
  - Added `zone_id` field to the property list and form view (under Property Specifications).
  - Added search by zone and group-by zone filters to the property search view.
- Added access rules for the new models in `ir.model.access.csv` to allow all portal/standard users full read/write access.
