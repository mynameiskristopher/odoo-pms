# Changelog - 2026-07-14

## [Added] PMS Zones Hierarchy Inheritance and Amenities Override System

- **Created Property Amenity model (`pms.amenity`)**:
  - Allows defining amenities like WiFi, Pool, and Parking.
  - Linked to both zones and properties.
- **Implemented Recursive Configuration & Amenities Inheritance on PMS Zones (`pms.zone`)**:
  - Zones can inherit amenities recursively from parent zones.
  - Added `override_amenities` (Boolean) and `custom_amenity_ids` (Many2many) to override parent amenities (including explicitly clearing amenities).
  - Added `checkin_time` and `checkout_time` float override fields on `pms.zone` which inherit parent values if not set.
  - Added `pets_allowed` selection override field on `pms.zone` (`yes`, `no`, `inherit`) inheriting parent settings if set to `inherit`.
  - Added recursive computed and stored fields for resolved configurations: `effective_checkin_time`, `effective_checkout_time`, `effective_pets_allowed`, and `amenity_ids`.
- **Implemented Inheritance & Overriding on Vacation Rental Properties (`pms.property`)**:
  - Property inherits effective logistics configuration (`checkin_time`, `checkout_time`, `pets_allowed`) and effective amenities (`amenity_ids`) from its assigned `zone_id`.
  - Added `override_config` (Boolean) to allow properties to override zone logistics.
  - Added `override_amenities` (Boolean) and `custom_amenity_ids` (Many2many) to allow properties to override zone amenities.
- **Added Views & Access Rules**:
  - Created list and form views for `pms.amenity`.
  - Updated zone and property form views to group and expose override options versus effective/resolved read-only settings.
  - Registered access control rules for `pms.amenity`.
- **Created Unit Tests (`tests/test_zone_hierarchy.py`)**:
  - Formulated recursive inheritance tests validating correct resolution of checkin, checkout, pets allowed, and amenities across three-level zone hierarchies and associated properties.
