# Changelog - 2026-06-23

## [Added] Property Listing Details, Address, and Capacity Fields

- Added 16 new fields to `pms.property` model to support listing metadata, detailed capacity, addresses, and compliance details:
  - Listing identifiers & teaser: `short_name` (Char), `unit_code` (Char), `headline` (Char), `short_description` (Text), `directions` (Text).
  - Address details matching Odoo standard format: `street` (Char), `street2` (Char), `city` (Char), `state_id` (Many2one to `res.country.state`), `zip` (Char), `country_id` (Many2one to `res.country`).
  - Capacity & Bathroom details: `bedrooms` (Integer), `full_bathrooms` (Integer), `three_quarter_bathrooms` (Integer), `half_bathrooms` (Integer).
  - Compliance: `municipality_id` (Char).
- Updated property form view (`view_pms_property_form`) to include the new fields:
  - Displayed `headline` under the title area in `h2`.
  - Added capacity features (`bedrooms`, bathroom counts in an inline row layout) and code/name identifiers inside "Property Specifications" group.
  - Reorganized "Logistics & Location" into "Logistics & Rent" and created a new notebook tab "Location & Compliance" hosting address fields, coordinates, permit numbers (`municipality_id`), and `directions`.
- Updated Schema.org mapping in `pms_property.md` documentation to cover the new fields.
