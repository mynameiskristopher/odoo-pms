# Changelog - 2026-07-17

## [Added] Owner Relationship Manager (ORM) Pipeline

- Created `pms.owner.lead` model for a dedicated owner pipeline — completely separate from Odoo's CRM (`crm.lead`).
  - Inherits `mail.thread` and `mail.activity.mixin` for chatter, activities, and email integration.
  - Pipeline stages with `pms.owner.lead.stage` model: New, Contacted, Negotiating, Onboarded (won), Inactive (lost).
  - Kanban pipeline view with drag-and-drop stage transitions, priority widget, and avatar assignment.
  - Fields: name, partner_id, owner_id, user_id, stage_id, priority, expected_property_count, expected_monthly_revenue, date_deadline, date_opened, date_closed, kanban_state, tag_ids, description.
  - Won/Lost actions with automatic date_closed tracking.
  - Auto-fills partner_id from owner_id when an owner is selected.
  - `pms.owner.lead.tag` model for pipeline-specific tagging.
- Created views: kanban (pipeline), list, form, search, stage config, and tag config.
- Added menu: Owners → Owner Pipeline + Owners → Configuration → Pipeline Stages / Lead Tags.
- Created `pms_orm_data.xml` with 5 default pipeline stages and an "Owner" partner category.
- Added ACLs for `pms.owner.lead`, `pms.owner.lead.stage`, and `pms.owner.lead.tag`.

## [Changed] Owner Model Refactored to Link with res.partner

- `pms.owner` now requires and auto-creates a `res.partner` (contact) record on creation.
  - `partner_id` Many2one to `res.partner` (required, ondelete='restrict').
  - `create()` override auto-creates a partner with the owner's name, email, phone, website, and address.
  - Partner is auto-tagged with "Owner" partner category for filtering in Contacts.
  - `write()` syncs `owner_type` to partner's `is_company` field.
  - Smart button on owner form to open the linked contact.
- Removed duplicate identity/contact fields from `pms.owner` (now delegated to partner):
  - `name` → stored related field from `partner_id.name`
  - `email`, `phone`, `website` → related fields from partner
  - `street`, `street2`, `city`, `state_id`, `zip`, `country_id` → related fields from partner
  - Removed: `street_address`, `extended_address`, `locality`, `region`, `postal`, `country` (raw text fields replaced by proper Odoo address fields via partner)
  - Removed: `fax` (not commonly used; can be added to partner if needed)
- Tax address fields kept on `pms.owner` (separate from business address on partner): `tax_street_address`, `tax_extended_address`, `tax_locality`, `tax_region`, `tax_postal_code`, `tax_country`, `tax_phone`.
- Removed constraints for old `country` field (now using `country_id` Many2one via partner — no need for manual 2-char ISO validation).
- `owner_type` expanded from single-option ('owner') to ('individual', 'company') for proper partner type sync.
- Updated owner views to use partner-related fields for address and contact info.
- Added "Pipeline" tab on owner form showing related leads.
- All downstream models (`pms.unit.owner`, `pms.owner.agreement`, `pms.owner.settlement`) continue to work unchanged — they reference `pms.owner` which now delegates identity to `res.partner`.

## [Added] Owner Partner Category

- Created `res.partner.category` record "Owner" (`partner_category_owner`) for tagging owner contacts.
- Owners auto-tagged on creation so they can be filtered in Odoo's built-in Contacts app.
